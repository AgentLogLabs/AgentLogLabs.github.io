# 使用教程

本文将引导您完成 AgentLog 的常见使用场景。

## 场景一：记录一次 AI 会话

### 准备工作
1. 确保 AgentLog 后台服务已启动（默认端口 7892）
2. 已配置 AI Agent 的 MCP 连接（参见 [MCP 集成指南](./mcp-integration)）

### 操作步骤
1. 在 VS Code 中打开您的项目
2. 使用 Cline 或 OpenCode 进行代码对话，例如：
   ```
   用户：请帮我重构 src/utils.ts 中的 calculateTotal 函数
   ```
3. AI Agent 会自动通过 MCP 协议上报交互记录
4. 在 VS Code 侧边栏的 AgentLog 面板中查看新会话

### 验证记录
- 侧边栏的「未提交的 AI 会话」区域应显示新会话
- 点击会话可查看完整的 Prompt、Response 和推理过程

## 场景二：查看并绑定到 Git Commit

### 自动绑定
1. 完成代码修改后，执行 Git 提交：
   ```bash
   git add .
   git commit -m "重构 calculateTotal 函数"
   ```
2. AgentLog 的 Git Hook 会自动将当前工作区的游离会话绑定到本次 Commit
3. 侧边栏的「历史 Commit 记录」区域会显示新 Commit 及其关联会话

### 手动绑定
如果自动绑定未生效，可手动操作：
1. 在侧边栏右键点击游离会话
2. 选择「绑定到最新 Commit」或「选择 Commit 绑定」
3. 选择目标 Commit 完成绑定

## 场景三：导出周报/PR 说明

### 导出周报
1. 在 AgentLog 侧边栏点击「导出」按钮
2. 选择「中文周报」格式
3. 配置时间范围（如本周、本月）
4. 点击「生成」并保存 Markdown 文件

### 导出 PR 说明
1. 在侧边栏找到目标 Commit 节点
2. 右键点击，选择「生成 PR 说明」
3. 复制生成的 Markdown 文本
4. 粘贴到 GitHub/GitLab 的 PR 描述中

## 场景四：复活上下文（核心功能）

### 什么是上下文复活？
当您或团队成员需要继续处理之前由 AI 辅助的代码时，「上下文复活」功能可以一键提取当时的 AI 交互记录，作为新对话的上下文。

### 操作步骤
1. 在侧边栏找到历史 Commit 节点并展开
2. 找到目标 AI 会话，点击「查看详情」
3. 在详情页点击「复活上下文」按钮
4. AgentLog 会将以下内容复制到剪贴板：
   ```
   【历史 AI 上下文复活】
   
   原始任务：重构用户认证模块
   历史推理：分析原函数职责...
   AI 回应：我将帮你重构...
   
   涉及文件：
   - src/auth.ts
   - src/middleware.ts
   
   请基于此上下文继续工作...
   ```
5. 将剪贴板内容粘贴到 Cline/Cursor/OpenCode 的聊天框中
6. 新 AI 对话将继承历史上下文，减少重复解释

### 使用场景
- **交接项目**：新成员快速了解之前的 AI 辅助决策
- **中断后继续**：几天后继续未完成的任务，恢复记忆
- **代码审查**：Reviewer 理解 AI 辅助的修改意图

## 场景五：多项目管理

### 配置多个工作区
AgentLog 自动识别工作区路径，支持同时监控多个项目：

1. 打开第一个项目文件夹
2. AgentLog 自动切换到该工作区上下文
3. 所有会话记录都会关联到当前工作区路径
4. 切换到另一个项目文件夹时，AgentLog 自动切换上下文

### 工作区筛选
在侧边栏顶部可使用筛选器：
- 按工作区路径过滤会话
- 查看所有工作区的聚合统计
- 导出时按工作区选择数据

## 场景六：团队协作

### 数据共享
虽然 AgentLog 是本地优先工具，但支持团队协作：

1. **导出共享**：将 JSONL 格式的原始数据导出并分享给团队成员
2. **导入查看**：团队成员可导入他人的数据在本地查看
3. **统一配置**：团队使用相同的 MCP 配置，确保数据格式一致

### 代码审查集成
在 PR 描述中包含 AI 交互上下文：

1. 使用 AgentLog 生成 PR 说明
2. 将生成的 Markdown 粘贴到 PR 描述
3. Reviewer 可点击链接查看详细的 AI 交互记录
4. 减少沟通成本，提高审查效率

## 最佳实践

### 记录完整性
- 使用支持 MCP 的 AI Agent，确保完整记录推理过程
- 对于不支持 MCP 的工具，考虑手动补充记录
- 重要的架构决策，建议在 AI 对话后添加人工备注

### Commit 绑定时机
- 完成一个逻辑完整的任务后及时提交并绑定
- 避免长时间累积大量游离会话
- 提交前检查会话绑定状态

### 数据管理
- 定期导出重要项目的完整数据作为备份
- 使用数据保留策略清理过期数据
- 敏感项目可配置独立的数据库路径

### 团队规范
- 统一团队的 AI Agent 配置
- 建立 PR 描述中包含 AI 上下文的规范
- 新成员入职时配置好 AgentLog 环境

## 故障排除

### 常见问题
1. **会话未显示**：检查 MCP 配置，重启 AI Agent
2. **绑定失败**：验证 Git Hook 是否安装，查看后端日志
3. **导出异常**：检查文件写入权限，确认数据量是否过大

### 获取帮助
- 查看 [常见问题](./faq) 页面
- 检查 AgentLog 扩展日志（VS Code 输出面板）
- 提交 [GitHub Issue](https://github.com/agentlog/agentlog/issues) 获取技术支持
